プライバシーポリシー

iOSアプリ「読書空間」（以下「本アプリ」）における、利用者情報の取り扱いについて定めます。

最終更新日：2026年7月5日

1基本方針

本アプリは、利用者が記録した読書データを端末内にのみ保存します。アカウント登録は不要で、開発元（fun-products）は利用者の読書データを収集・保管しません。

本アプリは広告の表示および広告目的のトラッキングを行いません。

利用状況の把握と品質改善のため、Firebase Analyticsを使用します。Firebase Analyticsで収集される情報は、広告配信、広告ターゲティング、他社アプリ・Webサイトを横断した追跡には使用しません。

2端末内に保存する情報

本アプリでは、次の情報を利用者の端末内に保存します。これらは開発元のサーバーへ送信されません。

  • 本棚の登録内容（タイトル、著者、ISBN、出版社などの書誌情報）
  • 読書状態・所有状態
  • メモ、評価、読み始めた日・読了日
  • 表紙画像のキャッシュ

3外部サービスへの送信

本アプリは、本を検索するときに、利用者が入力した検索語のみを外部の書誌情報サービスへ送信します。書籍の表紙画像を取得するために、これらのサービスへ通信する場合があります。

本棚の登録内容、メモ、評価、読み始めた日・読了日、バックアップ（JSON）の内容は、外部サービスへ送信しません。

利用する外部サービス

  • 楽天ブックス書籍検索API（楽天ウェブサービス）
  • Google Books API
  • 国立国会図書館サーチ（NDL Search）
  • openBD

これらのサービスにおける情報の取り扱いは、各サービスのプライバシーポリシーに従います。

4カメラの利用

本アプリは、本のISBNバーコードを読み取って登録するためにカメラを使用します。バーコードの読み取りは端末内で処理され、カメラの映像や撮影画像を保存・外部送信することはありません。

カメラの利用は、利用者が許可した場合にのみ行われます。

5データの保存・削除・バックアップ

読書データは端末内に保存され、本アプリを削除すると端末から削除されます。

機種変更やアプリ削除に備えて、設定画面からJSON形式でデータをエクスポート（書き出し）・インポート（読み込み）できます。エクスポートしたファイルの保存先・共有方法は利用者が管理します。

6利用状況の計測

本アプリでは、アプリの利用状況を把握し、機能改善や不具合調査に役立てるため、Firebase Analyticsを使用します。

Firebase Analyticsにより、アプリの起動、画面表示、機能の利用状況、端末の種類、OS、概略地域、アプリ内購入に関する情報、アプリインスタンス識別子などが収集される場合があります。

本アプリでは、書名、著者名、ISBN、検索キーワード、メモ本文など、利用者の読書内容そのものをFirebase Analyticsのイベントとして送信しない設計にしています。

Firebase Analyticsで収集される情報の取り扱いについては、GoogleおよびFirebaseの説明もあわせてご確認ください。

  • Privacy and Security in Firebase
  • Firebase Help: Data collection
  • Google プライバシーポリシー

7広告識別子とトラッキング

本アプリでは、IDFA（広告識別子）の取得、AdSupport.frameworkの利用、AppTrackingTransparencyによる広告目的のトラッキング許可要求は行いません。

Firebase Analyticsで収集される情報は、広告配信、広告ターゲティング、他社アプリ・Webサイトを横断したユーザー追跡、データブローカーへの共有には使用しません。

8子どものプライバシー

本アプリは特定の年齢層を対象とせず、個人を特定する情報の登録を求めません。読書データはすべて端末内に保存されます。

9本ポリシーの変更

本アプリの機能追加や関連法令の変更に伴い、本ポリシーを更新することがあります。重要な変更がある場合は、本ページにて告知します。
